# 外网访问超时 - 排查与解决

## 当前状态

✅ **DNS 解析正常**：`home.qlsm.net` → `113.250.72.224`  
✅ **本地服务正常**：`https://127.0.0.1` 可访问  
❌ **外网访问超时**：`https://home.qlsm.net` 无法连接

---

## 原因分析

外网无法访问的常见原因：

1. **路由器端口映射未配置**（最常见）
2. **运营商封禁 80/443 端口**（中国移动/联通/电信常见）
3. **路由器防火墙阻止**
4. **公网 IP 不是真实公网 IP**（运营商级 NAT/CGNAT）

---

## 解决方案

### 方案 1：配置路由器端口映射（首选）

**步骤**：

1. 登录路由器管理界面（通常是 `http://192.168.1.1`）
2. 找到「端口映射」/「虚拟服务器」/「NAT 设置」
3. 添加规则：
   - **外部端口 80** → **内部 IP 192.168.1.100:80**
   - **外部端口 443** → **内部 IP 192.168.1.100:443**
4. 保存并重启路由器（如需要）

**验证**：配置后从外网（手机 4G/5G）访问 `https://home.qlsm.net`

---

### 方案 2：使用非标准端口（如果 80/443 被封）

如果运营商封禁了 80/443，改用其他端口（如 8443）：

#### 2.1 修改 nginx 配置

```bash
sudo nano /etc/nginx/sites-available/openclaw
```

修改为监听 8443：

```nginx
server {
    listen 8443 ssl http2;
    server_name home.qlsm.net;
    # ... 其他配置保持不变
}
```

#### 2.2 路由器映射

- **外部端口 8443** → **内部 IP 192.168.1.100:8443**

#### 2.3 访问

```
https://home.qlsm.net:8443/?token=你的token
```

---

### 方案 3：Cloudflare Tunnel（无需端口映射）

如果路由器不支持端口映射或运营商封禁端口，使用 Cloudflare Tunnel：

1. 安装 `cloudflared`：
   ```bash
   wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
   sudo mv cloudflared-linux-amd64 /usr/local/bin/cloudflared
   sudo chmod +x /usr/local/bin/cloudflared
   ```

2. 登录 Cloudflare：
   ```bash
   cloudflared tunnel login
   ```

3. 创建隧道：
   ```bash
   cloudflared tunnel create openclaw
   ```

4. 配置路由：
   ```bash
   cloudflared tunnel route dns openclaw home.qlsm.net
   ```

5. 运行隧道：
   ```bash
   cloudflared tunnel run openclaw
   ```

这样无需端口映射，Cloudflare Tunnel 会建立加密连接。

---

### 方案 4：检查是否为 CGNAT

某些运营商使用 CGNAT，你的「公网 IP」实际是内网 IP。

**检查方法**：

```bash
# 查看路由器 WAN IP
# 登录路由器，查看「网络状态」或「WAN 信息」
# 如果 WAN IP 与 curl checkip.amazonaws.com 返回的 IP 不同，可能是 CGNAT
```

**解决**：联系运营商申请**公网 IP**（可能需要付费或说明用途）。

## 当前建议

1. **先配置路由器端口映射**（方案 1）
2. **如果仍超时**，检查运营商是否封禁端口
3. **如果端口被封**，使用非标准端口（方案 2）或 Cloudflare Tunnel（方案 3）
