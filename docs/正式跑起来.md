# 正式跑起来 - 完整步骤

按顺序执行，直到能在终端或 Telegram 里和 AI 对话。

---

## 第一步：环境准备（只需做一次）

在 **WSL** 里执行：

```bash
# 1. 进入项目
cd /home/cool/universal-ai-assistant

# 2. 若未装 Node.js 22+，先装（二选一）
#    curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
#    sudo apt install -y nodejs

# 3. 若未装 Docker，先装
#    curl -fsSL https://get.docker.com | sudo sh
#    sudo usermod -aG docker $USER
#    然后退出终端重新登录

# 4. 执行安装脚本
sed -i 's/\r$//' install.sh start.sh 2>/dev/null
chmod +x install.sh start.sh
./install.sh
```

---

## 第二步：配置 CLIProxyAPI（拿 API Key）

1. **启动 CLIProxyAPI**（若第一步已自动启动可跳过）：
   ```bash
   cd /home/cool/universal-ai-assistant
   docker compose up -d
   ```

2. **打开管理面板**：浏览器访问  
   **http://localhost:8317/management.html**  
   （若打不开，试 http://本机IP:8317/management.html）  
   若提示输入**管理密钥**，填：`openclaw-management`。  
   若提示**「访问被拒绝，权限不足」**：先确认已输入上述密钥；若从 Windows 浏览器访问 WSL，需在配置里把 `remote-management.allow-remote` 设为 `true`，覆盖配置后执行 `docker compose restart`（见项目根目录 `config/cliproxyapi.yaml.example`）。

3. **添加账号**：在面板里添加 Claude Code（或 Gemini 等）账号，按页面提示完成 OAuth 登录。

4. **复制 API Key**：在配置/API Key 里复制一个 Key，后面填到 OpenClaw 用。

---

## 第三步：配置 OpenClaw（填 API Key）

在 WSL 里执行：

```bash
cd /home/cool/universal-ai-assistant
python3 scripts/config_wizard.py
```

按提示输入：
- **CLIProxyAPI baseURL**：直接回车（默认 `http://localhost:8317/v1`）
- **CLIProxyAPI API Key**：粘贴第二步复制的 Key（必填）
- Telegram 相关：不需要就直接回车跳过

### （可选）国内网络：给 WSL 里的 Gateway 配代理

如果你在国内使用 **Telegram / OpenRouter**，常见现象是 Gateway 日志出现 `TypeError: fetch failed`（无法访问外网）。

本项目支持在启动时自动加载项目根目录的 `.env`（见 `.env.example`），推荐这样做：

```bash
cd /home/cool/universal-ai-assistant
cp .env.example .env
nano .env
```

- **WSL2 NAT 模式**下，Windows 上的 `localhost:7890` 这类代理端口在 WSL 里通常不可直接用；需要把 `127.0.0.1` 换成 **Windows 宿主机 IP**（通常可从 `/etc/resolv.conf` 的 `nameserver` 获取）。
- 配好 `HTTP_PROXY/HTTPS_PROXY` 后，用 `./start.sh` 启动，Gateway 才会走代理访问 Telegram / OpenRouter。

或手动改配置：

```bash
nano ~/.openclaw/openclaw.json
# 把 models.providers.cliproxy.apiKey 换成第二步拿到的 API Key，保存退出
```

若运行 `openclaw agent` 时提示 **Config invalid / Legacy config keys**，执行一次迁移再试：

```bash
openclaw doctor --fix
```

## 第四步：启动 OpenClaw Gateway

在 WSL 里**保持一个终端常开**运行：

```bash
cd /home/cool/universal-ai-assistant
./start.sh
```

看到类似 “Gateway listening on …” 或无报错即表示已启动。**不要关这个终端**。

### （可选）Telegram 通道首次配对

如果你启用了 `channels.telegram`：

1. 在 Telegram 里给你的 Bot 发一句话（例如 `你好`）
2. Bot 会返回一段提示，包含 `Pairing code: XXXXXXXX`
3. 在跑 Gateway 的这台机器（WSL）上执行：

```bash
openclaw pairing approve telegram XXXXXXXX
```

提示成功后，再发消息给 Bot 即可开始对话。

## 第五步：首次对话（验证已跑起来）

**新开一个 WSL 终端**，执行：

```bash
openclaw agent --message "你好，请用一句话介绍你自己"
```

若返回一句 AI 的回复，说明已经**正式跑起来**。

再试指定 agent：

```bash

```

## 常见问题（快速定位）

### 1) `openclaw update` 权限不足

若提示无权写入 `/usr/lib/node_modules`，用：

```bash
sudo npm install -g openclaw@latest
```

### 2) WSL 提示 “检测到 localhost 代理配置，但未镜像到 WSL”

这是 WSL 的提示：Windows 的 `localhost` 代理在 WSL2 NAT 模式下通常不可直接用。请按本文上面的“给 WSL 里的 Gateway 配代理”章节，用 Windows 宿主机 IP 配置 `HTTP_PROXY/HTTPS_PROXY`，再启动 `./start.sh`。
