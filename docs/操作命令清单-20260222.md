# OpenClaw 操作命令清单（2026-02-22）

## 最优设置（当前状态）

| 项目 | 配置 |
|------|------|
| Gateway bind | loopback（仅 127.0.0.1，安全） |
| 外部访问 | 192.168.1.100:18790（port_forward 转发到 18789） |
| 本机访问 | 127.0.0.1:18789 |
| 环境变量 | OPENROUTER_API_KEY、OPENCLAW_GATEWAY_TOKEN 已写入 ~/.bashrc |
| systemd | 已注入 OPENROUTER_API_KEY、HTTP_PROXY、HTTPS_PROXY |

在 home 机器上 SSH 登录后，按顺序执行以下命令。

## 一、解决 MissingEnvVarError: OPENROUTER_API_KEY

**每次新开终端或 SSH 登录后，先执行：**

```bash
export OPENROUTER_API_KEY="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"
```

**或** 确保已写入 `~/.bashrc`（只需做一次）：

```bash
grep -q "OPENROUTER_API_KEY" ~/.bashrc || echo 'export OPENROUTER_API_KEY="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"' >> ~/.bashrc
source ~/.bashrc
```

## 四、Gateway 服务管理

```bash
# 查看状态
systemctl --user status openclaw-gateway

# 重启
systemctl --user restart openclaw-gateway

# 查看日志
tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log
```

---

## 五、可选：移除 Gateway 认证（恢复无 token 访问）

若仅本机使用、不需要 token 认证，可执行：

```bash
openclaw config unset gateway.auth.token
openclaw config unset gateway.auth.password
systemctl --user restart openclaw-gateway
```

**注意**：移除后 Control UI 和 CLI 将无需 token，安全性降低。

---

## 六、可选：把 key 直接写入 openclaw.json（不再依赖环境变量）

```bash
openclaw config set models.providers.openrouter.apiKey "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"
systemctl --user restart openclaw-gateway
```

## 七、其他 AI 建议中无效的命令（勿用）

| 命令 | 结果 |
|------|------|
| `openclaw device-token --rotate` | 命令不存在 |
| `openclaw config-validate` | 命令不存在 |
| `openclaw restart --all` | 命令不存在 |
| `openclaw install-service` | 命令不存在 |
| `openclaw config set gateway.logFile "..."` | 配置键不存在 |
| `openclaw gateway reset --reconfigure` | 选项不存在 |
| `openclaw gateway pair` | 需确认你当前版本是否支持 |
| `openclaw gateway start --pair-reset` | 需确认你当前版本是否支持 |
