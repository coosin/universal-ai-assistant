# 路由器端口映射设置指南（解决链接超时）

## 问题说明

家庭宽带通常使用 NAT，外网无法直接访问内网设备。需要配置**端口映射**（Port Forwarding）让外网流量能到达你的服务器。

---

## 设置步骤

### 1. 登录路由器管理界面

- 通常地址：`http://192.168.1.1` 或 `http://192.168.0.1`
- 用户名/密码：查看路由器背面标签，或咨询网络管理员

### 2. 找到端口映射/虚拟服务器设置

不同路由器界面名称可能不同：
- **端口映射** / **Port Forwarding**
- **虚拟服务器** / **Virtual Server**
- **NAT** / **NAT 设置**
- **高级设置** → **端口转发**

### 3. 添加端口映射规则

需要添加两条规则：

| 规则名称 | 外部端口 | 内部 IP | 内部端口 | 协议 |
|---------|---------|---------|---------|------|
| HTTP | 80 | 192.168.1.100 | 80 | TCP |
| HTTPS | 443 | 192.168.1.100 | 443 | TCP |

**示例配置**：
- **服务名称**：OpenClaw-HTTP
- **外部端口**：80
- **内部 IP**：192.168.1.100
- **内部端口**：80
- **协议**：TCP

重复添加 HTTPS（443）规则。

### 4. 保存并应用

保存设置后，某些路由器需要重启才能生效。

---

## 验证端口映射

配置完成后，从**外网**（手机 4G/5G 或朋友电脑）测试：

```bash
# 测试 HTTP（应重定向到 HTTPS）
curl -I http://home.qlsm.net

# 测试 HTTPS
curl -k https://home.qlsm.net/health
```

或直接在浏览器访问：`https://home.qlsm.net/?token=你的token`

## 常见问题

### 问题 1：找不到端口映射设置

- 某些路由器在「高级设置」或「防火墙」里
- 某些运营商路由器可能禁用了端口映射（需要联系运营商）

### 问题 2：配置后仍超时

1. **检查防火墙**：确保服务器防火墙允许 80/443（已配置）
2. **检查 nginx**：`sudo systemctl status nginx`
3. **检查 DNS**：`dig +short home.qlsm.net A` 应显示你的公网 IP
4. **检查公网 IP 变化**：家庭宽带 IP 可能变化，DDNS 会自动更新

### 问题 3：运营商封禁 80/443 端口

某些运营商（如中国移动/联通/电信）可能封禁 80/443 端口。解决方案：

- **方案 A**：使用非标准端口（如 8443），在路由器映射 8443 -> 192.168.1.100:443
- **方案 B**：使用 Cloudflare Tunnel（无需端口映射）
- **方案 C**：联系运营商申请开放端口（通常需要备案）

---

## 使用非标准端口（如果 80/443 被封）

如果运营商封禁了 80/443，可以改用其他端口：

1. **修改 nginx 配置**：
   ```nginx
   server {
       listen 8443 ssl http2;
       # ... 其他配置
   }
   ```

2. **路由器映射**：8443 -> 192.168.1.100:8443

3. **访问**：`https://home.qlsm.net:8443/?token=...`
