# OpenClaw 问题分析与修复总结（2026-02-22）

## 一、问题清单

| 问题 | 现象 | 根因 |
|------|------|------|
| **1. MissingEnvVarError: OPENROUTER_API_KEY** | `openclaw status` 等命令报错，无法读取配置 | `openclaw.json` 使用 `${OPENROUTER_API_KEY}`，但 **systemd 服务**和 **SSH 新 shell** 均未加载项目 `.env` |
| **2. gateway connect failed: pairing required** | CLI 无法连接 Gateway，`openclaw status` 显示 pairing required | 添加了 `gateway.auth.token` 和 `gateway.auth.password` 后，本地 CLI/Control UI 需携带 token 才能连接 |
| **3. OpenRouter 计费错误** | 模型调用返回「余额不足」 | OpenRouter API Key 额度用完或 key 无效 |
| **4. device token mismatch** | 设备令牌不匹配 | 更换 gateway auth 后，已配对设备需重新配对 |

---

## 二、根因分析

### 2.1 为何 OPENROUTER_API_KEY 经常「找不到」

- **项目 `.env`** 在 `~/universal-ai-assistant/.env`，内有 `OPENROUTER_API_KEY=sk-or-v1-...`
- **加载时机**：只有执行 `./start.sh` 时才会 `source .env`
- **未加载场景**：
  - `openclaw gateway` 通过 **systemd** 启动 → 不执行 `start.sh`，不加载 `.env`
  - 用户 SSH 登录后直接运行 `openclaw status` → 新 shell 未 source `.env`
- **结果**：`openclaw.json` 中的 `${OPENROUTER_API_KEY}` 无法被替换，报 `MissingEnvVarError`

### 2.2 为何出现 pairing required

- 其他 AI 执行了：
  ```bash
  openclaw config set gateway.auth.token $(openssl rand -hex 16)
  openclaw config set gateway.auth.password $(openssl rand -base64 12)
  ```
- 这为 Gateway 启用了 **token + password 认证**
- 本地 CLI（`openclaw status`）和 Control UI 访问 `http://127.0.0.1:18789/` 时，需携带 token 或完成配对
- 若未正确传 token，会报 `pairing required` 或 `gateway closed (1008): pairing required`

### 2.3 当前配置状态（简要）

- **openclaw.json**：`models.providers.openrouter.apiKey` 仍为 `${OPENROUTER_API_KEY}`（依赖环境变量）
- **systemd 服务**：`~/.config/systemd/user/openclaw-gateway.service` 中 **没有** `OPENROUTER_API_KEY`
- **gateway.auth**：已配置 token 和 password

---

## 三、推荐修复步骤

### 3.1 让 systemd 服务能读到 OPENROUTER_API_KEY

在 systemd 用户服务中注入环境变量。编辑 `~/.config/systemd/user/openclaw-gateway.service`，在 `[Service]` 的 `Environment=` 行后增加：

```ini
Environment=OPENROUTER_API_KEY=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
Environment=HTTP_PROXY=http://127.0.0.1:7890
Environment=HTTPS_PROXY=http://127.0.0.1:7890
```

然后执行：

```bash
systemctl --user daemon-reload
systemctl --user restart openclaw-gateway
```

### 3.2 让 SSH 登录后的 shell 有 OPENROUTER_API_KEY

已执行过 `echo "export OPENROUTER_API_KEY=\"...\"" >> ~/.bashrc` 的，新开终端会生效。  
若仍报错，可手动执行一次：

```bash
export OPENROUTER_API_KEY="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"
```

### 3.3 解决 pairing required（连接 Gateway）

**方式 A：用 token 访问 Control UI**

- 浏览器打开：`http://127.0.0.1:18789/?token=8f00672a94d1c43f1b74ef97a8fc2ef2`  
  （token 来自 `openclaw.json` 的 `gateway.auth.token`）

**方式 B：本地 CLI 使用 token**

- 设置环境变量：`export OPENCLAW_GATEWAY_TOKEN=8f00672a94d1c43f1b74ef97a8fc2ef2`
- 或查阅 `~/.openclaw/auth_credentials.txt`（若已按其他 AI 建议保存）

**方式 C：若仅本机使用、不需要认证**

- 可移除 gateway auth，恢复为无认证（会降低安全性）：
  ```bash
  openclaw config unset gateway.auth.token
  openclaw config unset gateway.auth.password
  systemctl --user restart openclaw-gateway
  ```

### 3.4 OpenRouter 计费错误

- 到 https://openrouter.ai/settings/credits 充值或确认 key 有效
- 或将 primary 模型改为不依赖 OpenRouter 的（如 `cliproxy/ds-v3`、`cliproxy/gemini-3-pro-preview`）

## 四、其他 AI 操作回顾

| 操作 | 效果 | 备注 |
|------|------|------|
| `export OPENROUTER_API_KEY=...` + `>> ~/.bashrc` | 新 shell 有该变量 | 对 systemd 无效 |
| `openclaw config set models.providers.openrouter.apiKey "sk-or-v1-..."` | 本应把 key 写入配置 | 当前 openclaw.json 仍为 `${OPENROUTER_API_KEY}`，可能被后续操作覆盖 |
| `openclaw config set gateway.auth.token/password` | 启用 Gateway 认证 | 导致 pairing required |
| `openclaw gateway install` + `start --force` | 安装并启动 systemd 服务 | 服务正常，但缺 OPENROUTER_API_KEY |
| `mv exa-tool /tmp/` | 移除有风险的 skill | 目录已不存在，无需再操作 |

## 五、建议的「一键修复」命令

在 home 机器上执行（按需调整 key）：

```bash
# 1. 为 systemd 服务注入 OPENROUTER_API_KEY 和代理
ENV_LINE='Environment=OPENROUTER_API_KEY=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'
grep -q "OPENROUTER_API_KEY" ~/.config/systemd/user/openclaw-gateway.service || \
  sed -i "/Environment=OPENCLAW_GATEWAY_PORT/a $ENV_LINE" ~/.config/systemd/user/openclaw-gateway.service

# 2. 重载并重启
systemctl --user daemon-reload
systemctl --user restart openclaw-gateway

# 3. 验证（需先 export OPENROUTER_API_KEY 或确保 .bashrc 已生效）
export OPENROUTER_API_KEY="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"
openclaw status
```

## 六、与项目 start.sh 的关系

- `./start.sh` 会 `source .env`，再执行 `openclaw gateway --port 18789`
- 适合**前台调试**，环境变量完整
- 若改用 **systemd** 管理 Gateway，必须把 `.env` 中需要的变量显式写入 service 的 `Environment=`，否则不会生效
